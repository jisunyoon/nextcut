# NextCut

실시간 상태 관리와 확장 가능한 UI 구조에 초점을 둔  
웹툰 추천·열람 웹 애플리케이션입니다.

---

## Overview

NextCut은 웹툰 열람 서비스에서 발생하는  
**복잡한 상태 흐름과 반복되는 UI 문제를 구조적으로 해결하기 위해** 설계한 프로젝트입니다.

웹툰 목록, 상세 열람, 사용자 인터랙션 과정에서  
전역 상태와 서버 상태, UI 상태를 명확히 분리하고  
유지보수와 확장성을 고려한 컴포넌트 구조에 집중했습니다.

---

## Tech Stack

- **React / TypeScript**  
  컴포넌트 기반 UI 설계 및 타입 안정성 확보

- **Zustand**  
  전역 상태 관리 단순화 및 예측 가능한 상태 흐름 설계

- **React Query (TanStack Query)**  
  서버 상태 관리 및 캐싱 전략 적용

- **Emotion**  
  컴포넌트 단위 스타일링 및 디자인 토큰 기반 테마 구성

- **Storybook**  
  UI 컴포넌트 분리 개발 및 문서화

- **WebSocket**  
  실시간 데이터 업데이트 처리

- **pnpm**  
  효율적인 패키지 및 의존성 관리

---

## Key Features

- 웹툰 목록 및 상세 열람 기능
- 사용자 인터랙션 기반 상태 관리
- 실시간 데이터 업데이트 반영
- 전역 상태와 UI 상태 분리
- 재사용 가능한 공통 UI 컴포넌트 구성
- 웹 접근성을 고려한 시맨틱 마크업 및 키보드 인터랙션 지원

---

## Architecture

- **Zustand**를 활용한 전역 상태 관리  
- **React Query**를 통한 서버 상태 분리
- UI 컴포넌트와 비즈니스 로직 분리
- 상태 변경 흐름을 예측 가능하게 설계
- 디자인 변경에 유연한 컴포넌트 구조

---

## Design System & Storybook

NextCut은 외부 UI 프레임워크(shadcn/ui 등)를 사용하지 않고,  
**Emotion 기반으로 필요한 컴포넌트만 직접 설계**하는 방식을 선택했습니다.

- 디자인 토큰(Color, Spacing, Typography) 기반 테마 구성
- Button, Input, Card, Modal 등 핵심 UI 컴포넌트 설계
- 컴포넌트 책임과 스타일 범위를 명확히 분리
- Storybook을 활용한 UI 컴포넌트 문서화 및 개발 환경 분리

이를 통해 빠른 구현보다 **확장성과 유지보수성을 고려한 UI 구조 설계 경험**에 집중했습니다.

---

## Why This Project

퍼블리셔와 프론트엔드 개발 경험을 바탕으로,  
단순히 화면을 구현하는 것을 넘어  
**상태 흐름과 구조를 고려한 UI 설계 역량을 강화하기 위해** 진행한 프로젝트입니다.

실무에서 자주 발생하는 반복적인 UI 수정과 상태 복잡도를  
프론트엔드 관점에서 구조적으로 해결하는 것을 목표로 했습니다.
